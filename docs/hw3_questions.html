Introduction

Blueprinty is a small software company that builds tools specifically for preparing and submitting patent applications to the US Patent and Trademark Office. Their marketing team wants to make a bold claim: engineering firms that use Blueprinty’s software are more likely to get their patents approved.

On the surface, this sounds like a question you could answer by simply comparing patent counts between customers and non-customers. But it isn’t that simple. Blueprinty’s customers are not chosen at random — firms that actively seek out specialized patent software are probably already more sophisticated, better resourced, or operating in regions with stronger innovation ecosystems. Any raw difference in patent counts between the two groups could easily reflect these pre-existing advantages rather than any effect of the software itself.

To study this more carefully, Blueprinty has shared data on 1,500 mature engineering firms, each with information on the number of patents they received over the last five years, their regional location, their age since incorporation, and whether they are a Blueprinty customer. Because the outcome is a count (non-negative integers), we model it using Poisson regression, estimated via Maximum Likelihood Estimation (MLE). This lets us hold age and region constant while estimating the association between software use and patent counts — a more credible comparison than looking at raw group means alone.

A Simple Poisson Model via Maximum Likelihood

Why Poisson?

Patent counts are non-negative integers: 0, 1, 2, 3, … A Normal distribution is a poor fit because it places probability mass on negative values and treats the data as continuous. The Poisson distribution is the natural starting point for non-negative integer outcomes. Its single parameter λ controls both the mean and the variance of the distribution, and the PMF is:

\[ f(Y \mid \lambda) = \frac{e^{-\lambda} \lambda^Y}{Y!}, \qquad Y = 0, 1, 2, \ldots \]

Deriving the Log-Likelihood

Given \(n\) independent observations \(Y_1, \ldots, Y_n\), the joint likelihood is:

\[ L(\lambda \mid \mathbf{Y}) = \prod_{i=1}^n \frac{e^{-\lambda} \lambda^{Y_i}}{Y_i!} \]

Taking the natural log turns the product into a sum (and makes optimization far easier numerically):

\[ \ell(\lambda \mid \mathbf{Y}) = \sum_{i=1}^n \left( -\lambda + Y_i \ln \lambda - \ln Y_i! \right) \]

\[ = -n\lambda + \ln\lambda \sum_{i=1}^n Y_i - \sum_{i=1}^n \ln Y_i! \]

Analytic MLE

Setting \(\frac{d\ell}{d\lambda} = 0\):

\[ \frac{d\ell}{d\lambda} = -n + \frac{1}{\lambda}\sum_{i=1}^n Y_i = 0 \implies \hat{\lambda}_{\text{MLE}} = \bar{Y} \]

This makes intuitive sense: the MLE for the Poisson rate is simply the sample mean, because \(\mathbb{E}[Y] = \lambda\) under the Poisson model.

Coding the Log-Likelihood

Code 
from scipy.special import gammaln  # log(Y!) = log(Gamma(Y+1))

def poisson_loglikelihood(lam, Y):
    """Log-likelihood of Poisson(lambda) for observed counts Y."""
    if lam <= 0:
        return -np.inf
    return np.sum(-lam + Y * np.log(lam) - gammaln(Y + 1))

Y = df["patents"].values

The numerical optimizer recovers the sample mean to five decimal places, exactly as the analytic derivation predicts. This cross-check confirms that the log-likelihood function is coded correctly before we move on to the regression setting.

Poisson Regression

From a Fixed Rate to Firm-Specific Rates

The simple Poisson model assumed every firm shares the same patent rate λ. That’s clearly unrealistic — firms differ in age, location, and technology adoption. Poisson regression lets each firm have its own rate:

\[ Y_i \sim \text{Poisson}(\lambda_i), \qquad \lambda_i = \exp(X_i'\beta) \]

We use the exponential link because \(\lambda_i\) must be strictly positive, while the linear predictor \(X_i'\beta\) can be any real number. The exponential function maps \(\mathbb{R} \to (0, \infty)\), keeping predictions valid.

Updating the Log-Likelihood

Code 
def poisson_regression_loglikelihood(beta, Y, X):
    """Poisson regression log-likelihood. lambda_i = exp(X_i @ beta)."""
    lam = np.exp(X @ beta)
    return np.sum(-lam + Y * np.log(lam) - gammaln(Y + 1))

Design choices:

  • Intercept: always included as the first column.
  • Age + Age²: firm age likely has a non-linear relationship with patent output — very young and very old firms may differ from those in their prime. Including the squared term lets the model capture this curvature.
  • Region dummies: we include indicators for all but one region (the dropped reference category). Keeping all regions would create perfect collinearity with the intercept, making the system unsolvable.
  • iscustomer: the key variable of interest.

Finding the MLE with scipy.optimize

Code 
from scipy.optimize import minimize
from scipy.linalg import inv

def neg_loglik(beta):
    eta = X @ beta
    mu = np.exp(eta)
    return -np.sum(Y * eta - mu - gammaln(Y + 1))

def neg_score(beta):
    """Gradient of the negative log-likelihood: X.T @ (mu - Y)"""
    mu = np.exp(X @ beta)
    return X.T @ (mu - Y)

def neg_hessian(beta):
    """Hessian of the negative log-likelihood: X.T @ diag(mu) @ X"""
    mu = np.exp(X @ beta)
    return X.T @ (mu[:, None] * X)

# Start intercept at log(sample mean) for numerical stability; other coeffs = 0
beta0 = np.r_[np.log(Y.mean()), np.zeros(X.shape[1] - 1)]

result_reg = minimize(
    neg_loglik,
    beta0,
    method="BFGS",
    jac=neg_score,
    options={"maxiter": 5000, "gtol": 1e-8}
)

beta_hat = result_reg.x

# SE = sqrt of diagonal of inverse observed information matrix (analytic Hessian)
H = neg_hessian(beta_hat)
cov_hat = inv(H)
se_hat = np.sqrt(np.diag(cov_hat))

results_df = pd.DataFrame({
    "Coefficient": beta_hat,
    "Std. Error": se_hat,
    "z-value": beta_hat / se_hat
}, index=feature_names)

print(results_df.round(5))

Interpreting the Coefficients

Because \(\lambda_i = \exp(X_i'\beta)\), the Poisson regression coefficients are log-rate ratios. A unit increase in a predictor multiplies the expected patent count by \(e^\beta\).

iscustomer coefficient: 0.2076
Rate ratio exp(β):      1.2307
Percentage change:      23.1%
p-value:                1.8275e-11

Key takeaways:

  • Blueprinty Customer: the coefficient is positive and statistically significant (p < 0.001). Exponentiating it gives a rate ratio of roughly 1.23, meaning that — after controlling for age and region — Blueprinty customers have about 23% more expected patents than otherwise comparable non-customers.

  • Firm Age has a positive coefficient and Age² has a negative coefficient, tracing an inverted-U relationship: patent output rises with age up to a point, then flattens or declines as older firms may shift priorities away from R&D.

  • Regional dummies capture differences in patent productivity across geographies, holding other factors constant.

The Effect of Blueprinty’s Software

The coefficient on iscustomer tells us the multiplicative change in the expected patent rate. But it’s more useful to report the estimated effect in patent counts — a quantity that non-technical stakeholders can immediately understand.

We use the Average Treatment Effect (ATE) via counterfactual prediction:

  1. Create \(X_0\): set iscustomer = 0 for every firm.
  2. Create \(X_1\): set iscustomer = 1 for every firm.
  3. Predict \(\hat{y}_0 = \exp(X_0 \hat\beta)\) and \(\hat{y}_1 = \exp(X_1 \hat\beta)\).
  4. Report \(\frac{1}{n}\sum_i (\hat{y}_{1i} - \hat{y}_{0i})\).
Code 
beta_glm = glm_result.params.to_numpy()

cust_col = feature_names.index("iscustomer")
X0 = X.copy(); X0[:, cust_col] = 0
X1 = X.copy(); X1[:, cust_col] = 1

y_hat_0 = np.exp(X0 @ beta_glm)
y_hat_1 = np.exp(X1 @ beta_glm)

ate = np.mean(y_hat_1 - y_hat_0)
print(f"Estimated ATE: {ate:.4f} additional patents per firm over 5 years")
Estimated ATE: 0.7928 additional patents per firm over 5 years

After holding firm age and regional location constant, switching a firm from non-customer to customer status is associated with an average of 0.793 additional patents over a five-year window. Given a typical baseline of around 3–4 patents per firm, this represents a meaningful relative increase.

What This Analysis Does — and Doesn’t — Establish

This result is consistent with Blueprinty’s marketing claim, and the association survives controls for two important confounders (age and region). However, a few important caveats apply:

  • Observational, not experimental: firms were not randomly assigned to use Blueprinty’s software. Firms that chose to adopt it may differ from non-adopters in ways we cannot fully measure — internal R&D culture, management quality, or patent strategy.
  • No pre-treatment baseline: ideally we would observe each firm’s patent rate before and after adopting the software. The cross-sectional data here cannot rule out that high-patenting firms were simply more likely to become customers in the first place.
  • Omitted variables: the model controls for age and region but not for firm size, R&D spending, or industry sub-sector, all of which could confound the relationship.

In short: the analysis provides credible correlational evidence in support of Blueprinty’s claim, but a randomized experiment or a difference-in-differences design with panel data would be needed to make a causal claim.

The **Poisson distribution** is the natural starting point for non-negative integer outcomes. Its single parameter λ controls both the mean and the variance of the distribution, and the PMF is:

$$
f(Y \mid \lambda) = \frac{e^{-\lambda} \lambda^Y}{Y!}, \qquad Y = 0, 1, 2, \ldots
$$

### Deriving the Log-Likelihood

Given $n$ independent observations $Y_1, \ldots, Y_n$, the **joint likelihood** is:

$$
L(\lambda \mid \mathbf{Y}) = \prod_{i=1}^n \frac{e^{-\lambda} \lambda^{Y_i}}{Y_i!}
$$

Taking the natural log turns the product into a sum (and makes optimization far easier numerically):

$$
\ell(\lambda \mid \mathbf{Y}) = \sum_{i=1}^n \left( -\lambda + Y_i \ln \lambda - \ln Y_i! \right)
$$

$$
= -n\lambda + \ln\lambda \sum_{i=1}^n Y_i - \sum_{i=1}^n \ln Y_i!
$$

### Analytic MLE

Setting $\frac{d\ell}{d\lambda} = 0$:

$$
\frac{d\ell}{d\lambda} = -n + \frac{1}{\lambda}\sum_{i=1}^n Y_i = 0 \implies \hat{\lambda}_{\text{MLE}} = \bar{Y}
$$

This makes intuitive sense: the MLE for the Poisson rate is simply the sample mean, because $\mathbb{E}[Y] = \lambda$ under the Poisson model.

### Coding the Log-Likelihood

```{python}
from scipy.special import gammaln  # log(Y!) = log(Gamma(Y+1))

def poisson_loglikelihood(lam, Y):
    """Log-likelihood of Poisson(lambda) for observed counts Y."""
    if lam <= 0:
        return -np.inf
    return np.sum(-lam + Y * np.log(lam) - gammaln(Y + 1))

Y = df["patents"].values
```

### Visualizing the Log-Likelihood

### Numerical MLE with `scipy.optimize`


## Poisson Regression

### From a Fixed Rate to Firm-Specific Rates

The simple Poisson model assumed every firm shares the same patent rate λ. That's clearly unrealistic — firms differ in age, location, and technology adoption. Poisson regression lets each firm have its own rate:

$$
Y_i \sim \text{Poisson}(\lambda_i), \qquad \lambda_i = \exp(X_i'\beta)
$$

We use the **exponential link** because $\lambda_i$ must be strictly positive, while the linear predictor $X_i'\beta$ can be any real number. The exponential function maps $\mathbb{R} \to (0, \infty)$, keeping predictions valid.

### Updating the Log-Likelihood

```{python}
def poisson_regression_loglikelihood(beta, Y, X):
    """Poisson regression log-likelihood. lambda_i = exp(X_i @ beta)."""
    lam = np.exp(X @ beta)
    return np.sum(-lam + Y * np.log(lam) - gammaln(Y + 1))
```

### Constructing the Design Matrix

```{python}
# One-hot encode region, drop one category to avoid multicollinearity
region_dummies = pd.get_dummies(df["region"], drop_first=True).astype(float)

X_df = pd.DataFrame({
    "intercept": 1.0,
    "age": df["age"],
    "age_sq": df["age"] ** 2,
}).join(region_dummies).join(df[["iscustomer"]].astype(float))

X = X_df.values
feature_names = X_df.columns.tolist()
print("Design matrix columns:", feature_names)
print("Shape:", X.shape)
```

**Design choices:**

- **Intercept**: always included as the first column.
- **Age + Age²**: firm age likely has a non-linear relationship with patent output — very young and very old firms may differ from those in their prime. Including the squared term lets the model capture this curvature.
- **Region dummies**: we include indicators for all but one region (the dropped reference category). Keeping all regions would create perfect collinearity with the intercept, making the system unsolvable.
- **`iscustomer`**: the key variable of interest.

### Finding the MLE with `scipy.optimize`

```{python}
from scipy.optimize import minimize
from scipy.linalg import inv

def neg_loglik(beta):
    eta = X @ beta
    mu = np.exp(eta)
    return -np.sum(Y * eta - mu - gammaln(Y + 1))

def neg_score(beta):
    """Gradient of the negative log-likelihood: X.T @ (mu - Y)"""
    mu = np.exp(X @ beta)
    return X.T @ (mu - Y)

def neg_hessian(beta):
    """Hessian of the negative log-likelihood: X.T @ diag(mu) @ X"""
    mu = np.exp(X @ beta)
    return X.T @ (mu[:, None] * X)

# Start intercept at log(sample mean) for numerical stability; other coeffs = 0
beta0 = np.r_[np.log(Y.mean()), np.zeros(X.shape[1] - 1)]

result_reg = minimize(
    neg_loglik,
    beta0,
    method="BFGS",
    jac=neg_score,
    options={"maxiter": 5000, "gtol": 1e-8}
)

beta_hat = result_reg.x

# SE = sqrt of diagonal of inverse observed information matrix (analytic Hessian)
H = neg_hessian(beta_hat)
cov_hat = inv(H)
se_hat = np.sqrt(np.diag(cov_hat))

results_df = pd.DataFrame({
    "Coefficient": beta_hat,
    "Std. Error": se_hat,
    "z-value": beta_hat / se_hat
}, index=feature_names)

print(results_df.round(5))
```

## The Effect of Blueprinty's Software

The coefficient on `iscustomer` tells us the *multiplicative* change in the expected patent rate. But it's more useful to report the estimated effect in **patent counts** — a quantity that non-technical stakeholders can immediately understand.

We use the **Average Treatment Effect (ATE)** via counterfactual prediction:

1. Create $X_0$: set `iscustomer = 0` for every firm.
2. Create $X_1$: set `iscustomer = 1` for every firm.
3. Predict $\hat{y}_0 = \exp(X_0 \hat\beta)$ and $\hat{y}_1 = \exp(X_1 \hat\beta)$.
4. Report $\frac{1}{n}\sum_i (\hat{y}_{1i} - \hat{y}_{0i})$.

```{python}
beta_glm = glm_result.params.to_numpy()

cust_col = feature_names.index("iscustomer")
X0 = X.copy(); X0[:, cust_col] = 0
X1 = X.copy(); X1[:, cust_col] = 1

y_hat_0 = np.exp(X0 @ beta_glm)
y_hat_1 = np.exp(X1 @ beta_glm)

ate = np.mean(y_hat_1 - y_hat_0)
print(f"Estimated ATE: {ate:.4f} additional patents per firm over 5 years")
```

After holding firm age and regional location constant, switching a firm from non-customer to customer status is associated with an average of **`{python} f"{ate:.3f}"` additional patents over a five-year window**. Given a typical baseline of around 3–4 patents per firm, this represents a meaningful relative increase.

### What This Analysis Does — and Doesn't — Establish

This result is consistent with Blueprinty's marketing claim, and the association survives controls for two important confounders (age and region). However, a few important caveats apply:

- **Observational, not experimental**: firms were not randomly assigned to use Blueprinty's software. Firms that chose to adopt it may differ from non-adopters in ways we cannot fully measure — internal R&D culture, management quality, or patent strategy.
- **No pre-treatment baseline**: ideally we would observe each firm's patent rate *before* and *after* adopting the software. The cross-sectional data here cannot rule out that high-patenting firms were simply more likely to become customers in the first place.
- **Omitted variables**: the model controls for age and region but not for firm size, R&D spending, or industry sub-sector, all of which could confound the relationship.

In short: the analysis provides credible *correlational* evidence in support of Blueprinty's claim, but **a randomized experiment or a difference-in-differences design with panel data would be needed to make a causal claim**.
